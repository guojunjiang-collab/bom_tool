# 附件预览功能设计方案

## 一、功能目标

在图文档管理界面中，支持对附件进行在线预览，自动根据附件格式选择合适的预览技术：

| 附件格式 | 预览技术 | 说明 |
|---|---|---|
| `.stp` / `.step` | Three.js + GLTFLoader | 后端 STP → glTF 转换，前端 WebGL 渲染三维模型 |
| `.pdf` | PDF.js | 浏览器端渲染 PDF 文档 |
| 其他格式 | 提示不支持 | 弹窗提示"该文件格式暂不支持在线预览" |

---

## 二、入口位置

### 1. 图文档管理汇总表格

操作列增加「预览」按钮（在「下载」按钮左侧），无附件时不显示：

```
操作：[预览] [下载] [编辑] [删除]
```

### 2. 图文档详情弹窗

底部 footer「关闭」按钮左侧增加「预览」按钮，无附件时不显示：

```
[预览]  [关闭]
```

点击后均通过 `window.open()` 打开新的浏览器标签页进行预览。

---

## 三、预览页面架构

新建独立页面 `frontend/preview.html`，与主应用分离，全屏预览：

```
preview.html
├── Three.js (CDN)              ← WebGL 3D 渲染引擎
├── OrbitControls.js (CDN)      ← 三维模型交互控制（旋转/缩放/平移）
├── GLTFLoader.js (CDN)         ← glTF/glb 模型加载器
├── PDF.js (CDN)                ← PDF 文档渲染
├── css/preview.css             ← 预览页面样式（新建）
└── js/preview.js               ← 预览逻辑（新建）
```

### 为什么用独立页面

- STP 三维渲染需要全屏空间，弹窗太小
- 用户要求"弹出新的浏览器标签页"
- 独立页面不影响主界面状态
- 深色背景更适合三维模型和文档预览

---

## 四、预览流程

```
用户点击「预览」
    ↓
前端判断附件后缀
    ↓
┌─ .stp / .step ──→ window.open('preview.html?att_id=xxx&token=yyy&type=stp')
│                        ↓
│                   前端请求 /api/v2/attachments/{id}/gltf
│                   ↓
│                   后端返回 .glb 文件（流式）
│                   ↓
│                   Three.js GLTFLoader 加载 → WebGL 渲染
│                   支持鼠标旋转、缩放、平移、线框切换
│
├─ .pdf ──────────→ window.open('preview.html?att_id=xxx&token=yyy&type=pdf')
│                        ↓
│                   前端请求 /api/v2/attachments/{id}/stream
│                   ↓
│                   PDF.js 渲染 Canvas → 支持翻页、缩放
│
└─ 其他 ──────────→ UI.toast('该文件格式暂不支持在线预览')
```

## 五、后端设计

### 5.1 新增接口

#### 流式下载接口（替代原有 base64 下载）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/v2/attachments/{id}/stream` | 流式下载附件，直接返回二进制文件 |

- 返回 `FileResponse`，浏览器直接处理二进制流
- 比 base64 方式节省约 33% 传输量（无需编码）
- 所有现有下载功能统一迁移到此接口

#### glTF 文件接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/v2/attachments/{id}/gltf` | 获取 STP 对应的 .glb 文件 |

- 如果 glb 文件已存在，直接返回
- 如果不存在，触发 STP → glTF 转换后返回
- 返回 `FileResponse`，media_type 为 `model/gltf-binary`

### 5.2 STP 转换服务

新建 `backend/app/stp_converter.py`：

- **转换工具**：使用 `cad-to-gltf` 库将 STP 转为 glTF Binary (.glb)
- **转换时机**：上传 STP 文件后异步触发（不阻塞上传响应）
- **存储位置**：glb 文件与 STP 文件保存在同一目录，后缀改为 `.glb`
- **缓存机制**：如果 glb 文件已存在且比 STP 新，跳过转换
- **超时保护**：单次转换最多 120 秒

转换脚本 `backend/app/stp_to_gltf.py`：
- 独立 Python 脚本，通过 subprocess 调用
- 便于隔离依赖和单独调试

### 5.3 上传流程变更

```
上传 STP 文件
    ↓
保存到文件系统（原有逻辑不变）
    ↓
创建数据库记录（原有逻辑不变）
    ↓
【新增】异步调用 convert_stp_to_gltf()
    ↓
subprocess 调用 stp_to_gltf.py
    ↓
在 STP 同目录生成 .glb 文件
```

### 5.4 新增依赖

`backend/requirements.txt` 新增：

```
cad-to-gltf>=0.3.0
```

## 六、前端设计

### 6.1 流式下载改造

`frontend/js/file-uploader.js` 中 `downloadFile()` 函数：

```javascript
// 旧方式：base64 → Blob → 下载（内存占用大，传输慢）
// 新方式：fetch stream → Blob → 下载（内存占用小，传输快）

async function downloadFile(attachmentId, filename) {
  const response = await fetch(`/api/v2/attachments/${attachmentId}/stream`, {
    headers: { 'Authorization': 'Bearer ' + token }
  });
  const blob = await response.blob();
  // 触发下载
}
```

新增 `getFileBlobUrl()` 辅助函数，供预览页面使用。

### 6.2 预览页面功能

#### PDF 预览

- 使用 PDF.js 渲染到 Canvas
- 支持：上一页 / 下一页 / 页码跳转
- 支持：滚轮缩放
- 深色背景 + 文档居中展示

#### STP 三维预览

- 使用 Three.js + GLTFLoader 加载 .glb 文件
- OrbitControls 交互控制：
  - **旋转**：鼠标左键拖拽
  - **缩放**：鼠标滚轮
  - **平移**：鼠标右键拖拽
- 工具栏按钮：
  - 🔄 重置视角
  - 🔲 线框/实体切换
- 加载中显示 loading 动画
- 加载失败显示错误提示

#### 页面布局

```
┌─────────────────────────────────────────────────┐
│  📄 文件预览 - Amo系统              [📥] [✕]    │  ← 工具栏
├─────────────────────────────────────────────────┤
│                                                 │
│                                                 │
│              预览内容区域                          │
│         （PDF Canvas 或 3D WebGL）               │
│                                                 │
│                                                 │
├─────────────────────────────────────────────────┤
│          [◀]  1 / 5  [▶]                        │  ← PDF 翻页
│          [🔄 重置] [🔲 线框]                     │  ← 3D 控制
└─────────────────────────────────────────────────┘
```

### 6.3 权限控制

- 通过 URL 参数传递 `token`（JWT）
- 预览页面请求时携带 token 鉴权
- 未登录或 token 无效时显示错误提示

## 八、关键设计决策

| 决策项 | 结论 |
|---|---|
| 下载方式 | 统一改为流式下载（替代 base64），提升速度，减少内存占用 |
| STP 转换方式 | 后端转换（cad-to-gltf），前端只负责渲染 glTF |
| 转换时机 | 上传完成后异步触发，不阻塞上传响应 |
| glTF 存储 | 与 STP 同目录同文件名，后缀改为 .glb |
| 预览页面 | 独立 HTML 页面，新标签页打开，全屏预览 |
| 移动端适配 | 暂不适配 |
