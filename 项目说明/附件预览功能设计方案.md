# 附件预览功能设计方案

## 一、功能目标

在图文档管理界面中，支持对附件进行在线预览，自动根据附件格式选择合适的预览技术：

| 附件格式 | 预览技术 | 说明 |
|---|---|---|
| `.stp` / `.step` | Three.js + GLTFLoader | 后端 STP → glTF 转换，前端 WebGL 渲染三维模型 |
| `.pdf` | 浏览器原生 PDF 阅读器 | 流式加载，支持 Range 请求，边下载边显示 |
| 其他格式 | 提示不支持 | 弹窗提示"该文件格式暂不支持在线预览" |

---

## 二、入口位置

### 1. 图文档管理汇总表格

操作列增加「预览」按钮（在「下载」按钮左侧），无附件时不显示：

```
操作：[预览] [下载] [编辑] [删除]
```

### 2. 图文档详情弹窗

底部 footer「关闭」按钮左侧增加「预览」按钮，无附件时不显示：

```
[预览]  [关闭]
```

点击后均通过 `window.open()` 打开新的浏览器标签页进行预览。

---

## 三、预览流程

### 3.1 PDF 预览（流式加载）

```
用户点击「预览」
    ↓
前端判断附件后缀为 .pdf
    ↓
window.open('/api/v2/attachments/{id}/preview?token=yyy', '_blank')
    ↓
浏览器直接请求后端，流式加载 PDF
    ↓
浏览器原生 PDF 阅读器边下载边显示
    ↓
支持：跳转任意页、缩放、打印、下载
```

**优点：**
- **流式加载** — 浏览器原生支持，边下载边显示，无需等待全量下载
- **Range 请求** — 大文件可分段加载，跳转到任意页瞬间响应
- **零内存占用** — 不经过 JavaScript 内存，直接由浏览器处理
- **速度快** — 30MB 文件也能快速打开首页

### 3.2 STP 三维预览

```
用户点击「预览」
    ↓
前端判断附件后缀为 .stp/.step
    ↓
window.open('preview.html?att_id=xxx&token=yyy&type=stp', '_blank')
    ↓
新标签页加载 preview.html + preview.js
    ↓
前端请求 /api/v2/attachments/{id}/gltf
    ↓
后端返回 .glb 文件（流式）
    ↓
Three.js GLTFLoader 加载 → WebGL 渲染
    ↓
支持鼠标旋转、缩放、平移、线框切换
```

## 四、预览页面架构

### 4.1 PDF 预览

**不需要独立页面**，直接打开后端 API URL：

```
/api/v2/attachments/{id}/preview?token=xxx
```

后端返回 `Content-Disposition: inline`，浏览器自动用原生 PDF 阅读器显示。

### 4.2 STP 三维预览

新建独立页面 `frontend/preview.html`，与主应用分离，全屏预览：

```
preview.html
├── Three.js (CDN)              ← WebGL 3D 渲染引擎
├── OrbitControls.js (CDN)      ← 三维模型交互控制（旋转/缩放/平移）
├── GLTFLoader.js (CDN)         ← glTF/glb 模型加载器
├── css/preview.css             ← 预览页面样式
└── js/preview.js               ← 预览逻辑
```

### 为什么 STP 用独立页面

- 三维渲染需要全屏空间，弹窗太小
- 用户要求"弹出新的浏览器标签页"
- 独立页面不影响主界面状态
- 深色背景更适合三维模型预览

## 五、后端设计

### 5.1 新增接口

#### 流式下载接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/v2/attachments/{id}/stream` | 流式下载附件，需要 Authorization header |

#### 预览接口（支持 query token）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/v2/attachments/{id}/preview?token=xxx` | 预览附件，返回 `Content-Disposition: inline` |

- 支持 token 作为 query 参数传递（用于浏览器直接打开）
- 返回 `FileResponse`，`Content-Disposition: inline` 让浏览器内嵌显示
- 浏览器原生支持 Range 请求，可流式加载大文件

#### 下载接口（支持 query token）

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/v2/attachments/{id}/direct-download?token=xxx` | 下载附件，返回 `Content-Disposition: attachment` |

- 触发浏览器下载行为（显示下载进度）

#### glTF 文件接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/v2/attachments/{id}/gltf` | 获取 STP 对应的 .glb 文件 |

- 如果 glb 文件已存在，直接返回
- 如果不存在，触发 STP → glTF 转换后返回
- 返回 `FileResponse`，media_type 为 `model/gltf-binary`

### 5.2 STP 转换服务

新建 `backend/app/stp_converter.py`：

- **转换工具**：使用 `cad-to-gltf` 库将 STP 转为 glTF Binary (.glb)
- **转换时机**：上传 STP 文件后异步触发（不阻塞上传响应）
- **存储位置**：glb 文件与 STP 文件保存在同一目录，后缀改为 `.glb`
- **缓存机制**：如果 glb 文件已存在且比 STP 新，跳过转换
- **超时保护**：单次转换最多 120 秒

转换脚本 `backend/app/stp_to_gltf.py`：
- 独立 Python 脚本，通过 subprocess 调用
- 便于隔离依赖和单独调试

### 5.3 上传流程变更

```
上传 STP 文件
    ↓
保存到文件系统（原有逻辑不变）
    ↓
创建数据库记录（原有逻辑不变）
    ↓
【新增】异步调用 convert_stp_to_gltf()
    ↓
subprocess 调用 stp_to_gltf.py
    ↓
在 STP 同目录生成 .glb 文件
```

### 5.4 新增依赖

`backend/requirements.txt` 新增：

```
cad-to-gltf>=0.3.0
```

## 六、前端设计

### 6.1 预览入口实现

`frontend/js/pages-documents.js` 中 `_previewDoc()` 函数：

```javascript
_previewDoc: function(fileId, fileName) {
  var ext = fileName.split('.').pop().toLowerCase();
  var token = localStorage.getItem('bom_api_token');
  
  if (ext === 'pdf') {
    // PDF 直接用浏览器原生阅读器流式加载
    var previewUrl = '/api/v2/attachments/' + fileId + '/preview?token=' + encodeURIComponent(token);
    window.open(previewUrl, '_blank');
  } else if (ext === 'stp' || ext === 'step') {
    // STP 使用自定义预览页面
    var previewUrl = '/preview.html?att_id=' + fileId + '&token=' + encodeURIComponent(token) + '&type=stp';
    window.open(previewUrl, '_blank');
  }
}
```

### 6.2 下载实现

```javascript
_downloadDoc: function(id) {
  var doc = Store.getById('documents', id);
  var token = localStorage.getItem('bom_api_token');
  
  fetch('/api/v2/attachments/' + doc.file_id + '/stream', {
    headers: { 'Authorization': 'Bearer ' + token }
  })
  .then(response => response.blob())
  .then(blob => {
    var url = URL.createObjectURL(blob);
    var a = document.createElement('a');
    a.href = url;
    a.download = doc.file_name;
    a.click();
  });
}
```

### 6.3 STP 预览页面功能

- 使用 Three.js + GLTFLoader 加载 .glb 文件
- OrbitControls 交互控制：
  - **旋转**：鼠标左键拖拽
  - **缩放**：鼠标滚轮
  - **平移**：鼠标右键拖拽
- 工具栏按钮：
  - 🔄 重置视角
  - 🔲 线框/实体切换
- 加载中显示 loading 动画
- 加载失败显示错误提示

## 八、关键设计决策

| 决策项 | 结论 |
|---|---|
| PDF 预览方式 | 浏览器原生阅读器 + 流式加载，不用 PDF.js |
| PDF 预览 URL | `/api/v2/attachments/{id}/preview?token=xxx` |
| 下载方式 | 统一改为流式下载（替代 base64），提升速度，减少内存占用 |
| STP 转换方式 | 后端转换（cad-to-gltf），前端只负责渲染 glTF |
| 转换时机 | 上传完成后异步触发，不阻塞上传响应 |
| glTF 存储 | 与 STP 同目录同文件名，后缀改为 .glb |
| STP 预览页面 | 独立 HTML 页面，新标签页打开，全屏预览 |
| 移动端适配 | 暂不适配 |

## 九、性能对比

### PDF 预览

| 指标 | 旧方案（PDF.js + blob） | 新方案（浏览器原生流式） |
|---|---|---|
| 30MB 文件首屏时间 | ~15 秒（全量下载） | ~1 秒（流式加载） |
| 内存占用 | 30MB+（blob 存内存） | ~0（浏览器管理） |
| 跳转任意页 | 需等待全量下载 | 瞬间（Range 请求） |
| 进度显示 | 无 | 浏览器原生 |
