**依据文档：** 《BOM版本对比功能规划文档.docx》

## 项目背景
现有BOM工具已具备基础的BOM对比功能，但对比《BOM版本对比功能规划文档》的要求，在可视化对比、自动变更识别、层级树对比、操作效率等方面存在较大差距。本项目旨在按照规划文档的规范，全面升级BOM对比功能，提供专业、直观、高效的版本差异分析工具。

## 🎯 第一阶段：需求分析与技术设计（0.5天）

### 1.1 文档深度解读
- 逐条分析规划文档中的功能点、视觉规范、操作流程
- 识别与现有BOM工具的差异点（当前对比功能可能较简单）

### 1.2 技术方案设计
- **API设计**：
  ```python
  POST /api/bom/compare
  {
    "left_assembly_id": "uuid",  # 左侧装配体版本ID
    "right_assembly_id": "uuid", # 右侧装配体版本ID
    "options": {
      "ignore_quantity": false,  # 是否忽略数量差异
      "max_depth": 10            # 最大对比深度
    }
  }
  ```
  响应结构包含变更分类（增、删、改、内部变更）、层级信息、差异详情。

- **前端组件结构**：
  - `BomComparePage`：主页面，管理状态
  - `VersionSelector`：左右版本选择器
  - `CompareTable`：双栏对比表格（支持虚拟滚动）
  - `DiffHighlighter`：差异高亮组件
  - `OperationToolbar`：操作工具栏

- **对比算法流程**：
  1. 预处理：拉取两个版本的BOM树，扁平化为带层级标记的列表
  2. 节点匹配：基于“层级+排序号+物料号”作为唯一键进行左右匹配
  3. 变更识别：按文档定义的4类变更（无、增、删、改、内部变更）分类
  4. 层级树重建：构建可展开/折叠的对比树结构

### 1.3 产出文档
- `BOM对比功能详细设计说明书.md`
- `API接口文档.md`
- `前端组件结构图`

## 🛠 第二阶段：后端对比算法API实现（1.5天）

### 2.1 数据库查询优化
- 编写递归CTE查询，快速获取装配体的完整BOM树（含层级、排序号）
- 添加索引优化：`bom_items(assembly_id, assembly_version)`

### 2.2 核心算法实现
- **预处理模块**：`bom/flatten.py`
  - 输入：装配体ID + 版本号
  - 输出：扁平化列表，每项包含 `level`, `sort`, `part_code`, `part_name`, `quantity`, `version` 等

- **匹配与对比模块**：`bom/compare.py`
  - 实现文档3.2节的变更识别逻辑
  - 内部变更检测：子项数量或顺序变化视为内部变更

- **API路由**：`routers/bom.py` 新增 `/compare` 端点
  - 输入验证（Pydantic Schema）
  - 调用对比算法，返回结构化差异数据
  - 错误处理与日志记录

### 2.3 测试与验证
- 单元测试：覆盖4种典型场景（文档第8章）
- 集成测试：模拟真实BOM数据，验证算法准确性
- 性能测试：千级节点BOM的对比响应时间 < 2s

### 2.4 产出物
- 可调用的 `/api/bom/compare` API
- 算法测试报告
- 性能基准数据

## 🎨 第三阶段：前端双栏对比界面开发（2天）

### 3.1 页面结构重构
- 修改 `BOM对比` 选项卡，替换现有简单对比界面
- 采用三栏布局（左版本选择 + 中操作栏 + 右版本选择）
- 下方为主对比区域（双栏表格 + 中缝提示）

### 3.2 对比表格组件
- **双栏同步滚动**：使用 `scroll` 事件同步左右滚动位置
- **层级缩进**：CSS `padding-left: calc(level * 2em)` 实现视觉层级
- **行高亮**：根据变更类型应用不同背景色
  - 无差异：白色
  - 新增：浅绿色 `#E6F4EA`，前缀 `+`
  - 删除：浅红色 `#FDEDED`，前缀 `-`
  - 修改：浅黄色 `#FFFBEB`，前缀 `✏️`
  - 内部变更：浅橙色 `#FFF3E0`，前缀 `?`

### 3.3 展开/折叠功能
- 每行左侧添加展开/折叠图标（▶/▼）
- 折叠时只显示父项，统计子项变更数量
- 状态持久化（localStorage记忆用户展开状态）

### 3.4 实时过滤
- 实现“只显示变更”开关
- 过滤逻辑：隐藏所有“无差异”行，保持层级关系

### 3.5 产出物
- 完整的双栏对比界面
- 响应式设计（支持不同屏幕宽度）
- 基础交互功能（展开/折叠、高亮、过滤）

## ⚙️ 第四阶段：操作功能与导出实现（1天）

### 4.1 操作工具栏
- **全展开/全折叠**：一键控制所有层级
- **变更定位**：
  - 上一个变更 / 下一个变更 按钮
  - 滚动到对应行并高亮闪烁
- **版本切换**：下拉菜单快速切换对比版本
- **刷新对比**：重新运行对比算法

### 4.2 导出功能
- **Excel导出**：使用 `xlsx` 库生成带格式的对比报告
  - 左右版本并排显示
  - 差异单元格着色
  - 包含汇总统计页
- **PDF导出**：使用 `jsPDF` + `html2canvas` 生成可打印报告
- 导出文件命名：`BOM对比_左版本_右版本_时间戳.xlsx`

### 4.3 用户偏好保存
- 记住用户最后对比的版本对
- 保存列宽、排序等界面状态
- 导出格式默认选择

### 4.4 产出物
- 功能齐全的操作工具栏
- Excel/PDF导出功能
- 用户偏好持久化
