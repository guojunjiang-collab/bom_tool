# 用户看板功能设计方案

## 一、功能定位

用户看板 = 每个用户的**个人工作空间**，核心是一个**自定义文件夹树**，文件夹内可以自由关联已有的零件、部件、图文档。支持**文件夹级共享**给同事。

---

## 二、数据模型设计

新增 4 张表：

### 1. `user_dashboards` — 看板主表（每用户一个）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 主键 |
| user_id | UUID → users.id | 所属用户，唯一 |
| name | VARCHAR(128) | 看板名称（默认"我的看板"） |
| created_at / updated_at | TIMESTAMP | |

### 2. `dashboard_folders` — 文件夹表（树形结构）

| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 主键 |
| dashboard_id | UUID → user_dashboards.id | 所属看板 |
| parent_id | UUID → self.id (nullable) | 父文件夹，null=根级 |
| name | VARCHAR(128) | 文件夹名称 |
| sort_order | INT | 同级排序（按名称自动排序） |
| created_at | TIMESTAMP | |

> 排序逻辑：前端/后端均按 `name` 字母序排列，无需手动调整。无限层级，通过 parent_id 自引用实现。

### 3. `dashboard_items` — 文件夹内容关联表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 关联记录 |
| folder_id | UUID → dashboard_folders.id | 所属文件夹 |
| entity_type | VARCHAR(16) | part / assembly / document |
| entity_id | UUID | 对应零件/部件/图文档 ID |
| created_at | TIMESTAMP | |

> 关联是**引用**，不复制数据。零件/部件/图文档本身不变，看板只是一个"快捷入口+分类视图"。排序按关联实体的 `name` 排列。

### 4. `dashboard_folder_shares` — 文件夹共享表

| 字段 | 类型 | 说明 |
|---|---|---|
| id | UUID | 主键 |
| folder_id | UUID → dashboard_folders.id | 被共享的文件夹 |
| shared_with_user_id | UUID → users.id | 被共享给谁 |
| permission | VARCHAR(16) | view（只读）/ edit（可编辑） |
| created_at | TIMESTAMP | |

> 共享是文件夹级别的，共享父文件夹不自动共享子文件夹，需要单独授权。查询时递归取所有有权限的文件夹。

## 三、API 设计

### 看板

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/dashboard` | 获取当前用户看板（含树 + 关联项 + 共享文件夹） |
| POST | `/api/dashboard/init` | 首次初始化看板（首次登录时自动调用） |

> 首次 `GET /api/dashboard` 时，如果用户没有看板，后端自动创建空看板返回。

### 文件夹

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/dashboard/folders` | 创建文件夹（传 parent_id） |
| PUT | `/api/dashboard/folders/{id}` | 重命名 / 移动到其他父文件夹 |
| DELETE | `/api/dashboard/folders/{id}` | 删除文件夹（级联删子文件夹 + 关联） |

### 关联项

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/dashboard/items` | 批量添加关联项到指定文件夹 |
| DELETE | `/api/dashboard/items/{id}` | 移除单个关联项 |

### 共享

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/dashboard/folders/{id}/shares` | 添加共享（传 user_id + permission） |
| GET | `/api/dashboard/folders/{id}/shares` | 查看某文件夹的共享列表 |
| DELETE | `/api/dashboard/folders/{id}/shares/{share_id}` | 取消共享 |

## 四、前端交互设计

### 导航栏

在现有侧边栏"仪表盘"下方新增一行，所有登录用户可见：

```
📊 仪表盘
🗂️ 用户看板   ← 新增
🔧 零件管理
📦 部件管理
📋 BOM管理
📄 图文档管理
👥 用户管理   (仅管理员)
📝 操作日志   (仅管理员)
⚙️ 系统设置   (仅管理员)
```

```
┌──────────────────────────────────────────────────┐

├───────────────┬──────────────────────────────────┤

│  │📁 设计  ⋮│ │  ┌────────────────────────────┐  │
│  │ └📁外壳 ⋮│ │  │ 编号  │名称  │版本│状态    │  │
│  │📁 采购  ⋮│ │  │ ...   │...  │... │...     │  │
│  │──────────│ │  └────────────────────────────┘  │

```

### 文件夹树交互

- 每个文件夹节点右侧显示 **"⋯"** 按钮（非右键菜单）
- 点击 ⋯ → 弹出操作菜单：
  - **重命名** → inline 编辑 / 小弹窗
  - **新建子文件夹** → 在当前文件夹下创建
  - **关联项目** → 快捷入口（等同于顶部"关联项目"按钮，自动选中当前文件夹）
  - **共享** → 弹窗选择用户 + 权限
  - **删除** → 二次确认
- 共享文件夹在树底部独立分区展示，带 📂 图标区分
- 树形展开/折叠，按名称自动排序
- 文件夹图标统一使用 📁

### 关联项目弹窗

```
┌──────────────────────────────────────┐
│  关联项目到「设计」                    │
│                                      │
│  筛选类型：[全部] [零件] [部件] [图文档] │
│  搜索：[_________________________]    │
│                                      │
│  ┌──────────────────────────────┐    │

│  └──────────────────────────────┘    │
│  (已在当前文件夹的项目自动置灰/排除)    │
```

- 顶部有类型筛选 Tab：**全部 / 零件 / 部件 / 图文档**
- 选择"全部"时，全局搜索三种类型
- 选择具体类型时，只在该类型中搜索
- 已在当前文件夹的项目自动置灰/排除，避免重复关联

### 详情查看

- 点击右侧表格中的任意一行 → **复用现有零件详情/部件详情/图文档详情弹窗**
- 不需要新建详情界面，根据 entity_type 调用对应管理模块已有的 `showDetail()` 方法

## 七、关键设计决策

| 决策项 | 结论 |
|---|---|
| 看板数量 | 每用户 1 个 |
| 关联深度 | 只关联顶层零件/部件/图文档，不递归展开 BOM 子项 |
| 权限 | 支持文件夹级共享（只读/可编辑） |
| 排序 | 按名称自动排序，不支持拖拽 |
| 文件夹图标 | 统一 📁 |
| 文件夹操作 | 使用"⋯"按钮弹出菜单，不支持右键 |
| 详情查看 | 复用现有零件/部件/图文档详情弹窗 |
| 关联搜索 | 支持类型筛选（全部/零件/部件/图文档）+ 搜索 |
