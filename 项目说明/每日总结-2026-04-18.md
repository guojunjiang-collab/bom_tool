- **后端**：schemas.py 添加 PartCreate/Update、AssemblyCreate/Update 附件字段

### 4. 附件GET返回空值问题
- **问题**：数据库写入正常但GET返回空，Pydantic序列化时 camelCase/snake_case 别名冲突
- **根因**：前端使用 `sourceFile`（camelCase），后端 schema 字段名为 `source_file`（snake_case），Pydantic 静默忽略不匹配字段

### 5. ⚠️ 编码灾难
- **根因**：PowerShell `Get-Content`/`Set-Content` 用 GBK 读取 UTF-8 文件再以 UTF-8 写回，双重编码损坏
- 中文数据不可逆丢失，多次修复尝试（GBK→UTF-8转换、字节级替换、反向转换）均失败
- 最终用2小时前备份恢复
- **关键教训**：绝不用 PowerShell 处理非 ASCII 文件；GBK 编码文件不做转码，只做 ASCII 字节级替换；HTML meta charset 与实际编码必须一致

- 改用 Git 操作，但遇 QClaw 安全策略拦截 C 盘操作

## 关键技术决策

1. **本地优先 + 后台静默同步**：保持 localStorage 为主数据库，每次 CRUD 后后台队列同步到后端
2. **附件独立存储**：附件 Base64 不再随实体保存到数据库，独立 attachments 表 + 按需加载
3. **编码规范**：绝不用 PowerShell 修改含中文文件，统一用 Python 二进制模式操作
4. **字段映射**：前端 camelCase → 后端 snake_case，通过 _fieldMap + toApi/fromApi 自动转换
