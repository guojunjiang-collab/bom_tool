# 零件状态变为草稿问题排查指南

## 问题描述

零件/部件状态为 `released`（发布）时，清除缓存再检出数据后，状态偶尔会变成 `draft`（草稿）。

---

## 排查步骤

### 阶段一：验证服务器数据（优先检查）

在服务器上执行以下命令，检查数据库和 API 返回是否正确：

```bash
# 1. 检查数据库中的状态
docker exec bom_postgres psql -U bomadmin -d bom_system -c "SELECT code, status FROM parts LIMIT 10;"

# 结果示例：
#      code      |  status
# ---------------+----------
#  V01-32100-001 | released
#  V01-32100-003 | draft
```

### 阶段二：部署调试版本

```bash
cd /opt/bom_tool
git pull
docker compose build backend
docker compose up -d --force-recreate
```

### 阶段三：复现问题并收集日志

1. 打开浏览器，进入开发者工具（F12）
2. 切换到 Console（控制台）标签
3. 清除缓存并检出数据
4. 问题复现后，查看控制台日志，查找以下关键信息：

| 日志关键词 | 含义 |
|-----------|------|
| `[Pull WARNING] converted.status is undefined!` | 服务器响应中没有 status 字段 |
| `[Pull WARNING] new item has undefined status!` | 新增时服务器没有返回 status |
| `[Pull] 覆盖本地 parts oldStatus=released -> newStatus=draft` | 这是问题发生的位置 |

---

## 问题可能原因

| 场景 | 可能原因 |
|------|---------|
| **偶尔发生** | 部分零件创建时后端未正确返回 status |
| **偶尔发生** | 某些特殊请求导致后端返回空 status |
| **偶尔发生** | 服务器曾重启，数据库连接问题导致部分字段缺失 |

---

## 常用命令速查

```bash
# 查看容器状态
docker compose ps

# 查看后端日志
docker compose logs -f backend

# 重启服务
docker compose restart

# 进入后端容器
docker exec -it bom_backend sh
```
