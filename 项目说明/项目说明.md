# BOM 管理工具 - 项目说明

## 一、项目概述
项目名称：网页版 BOM 管理工具
项目类型：前后端分离 Web 应用
版本：v1.0.0

### 1.1 核心目标
将原有的纯前端 BOM 管理工具升级为前后端分离架构，支持多用户、多角色、数据持久化存储。

### 1.2 技术栈
- 前端：原生 HTML/CSS/JS 单文件 SPA
- 后端：Python FastAPI（高性能 ASGI 框架）
- 数据库：PostgreSQL（持久化存储）
- 缓存：Redis
- 反向代理：Nginx
- 容器化：Docker + Docker Compose

## 四、API 接口说明

### 4.1 认证
  POST /api/auth/token          登录获取 Token（无需认证）
  GET  /api/auth/me             获取当前用户信息（需认证）
  POST /api/auth/change-password 修改当前用户密码（需认证）

### 4.2 用户管理（管理员）
  GET    /api/users/             获取用户列表
  POST   /api/users/             创建新用户
  GET    /api/users/{user_id}   获取用户详情
  PUT    /api/users/{user_id}   更新用户
  DELETE /api/users/{user_id}   删除用户

### 4.3 零件管理
  GET    /api/parts/             获取零件列表
  POST   /api/parts/             创建零件
  GET    /api/parts/{part_id}   获取零件详情
  PUT    /api/parts/{part_id}   更新零件
  DELETE /api/parts/{part_id}   删除零件

### 4.4 部件管理
  GET    /api/assemblies/        获取部件列表
  POST   /api/assemblies/        创建部件
  GET    /api/assemblies/{id}    获取部件详情
  PUT    /api/assemblies/{id}   更新部件
  DELETE /api/assemblies/{id}   删除部件

### 4.5 BOM 层级管理
  GET    /api/bom/tree/{type}/{id}  获取 BOM 树
  POST   /api/bom/items          创建 BOM 父子关系
  DELETE /api/bom/items/{item_id}  删除 BOM 关系

### 4.6 操作日志
  GET    /api/logs/              获取操作日志

## 五、数据库设计

### 5.1 数据模型
  users           用户表（用户名、姓名、角色、部门等）
  parts           零件表（编号、名称、规格、材料、单价、库存等）
  assemblies      部件表（编号、名称、规格、单价等）
  bom_items       BOM 关系表（父项类型/ID、子项类型/ID、用量）
  operation_logs  操作日志表（用户、操作类型、对象、详情、IP）

### 5.2 核心字段
  users: id(uuid), username, hashed, real_name, role, dept, phone, status, created, updated
  parts: id(uuid), code, name, spec, material, unit, price, supplier, stock, status, remark, created, updated
  assemblies: id(uuid), code, name, spec, unit, price, status, remark, created, updated
  bom_items: id(uuid), parent_type, parent_id, child_type, child_id, qty, created
  operation_logs: id(uuid), user_id, username, action, target_type, target_id, detail, ip, created

## 六、运维指南

### 6.1 常用命令
  停止服务：       docker-compose down
  清除数据（慎用）： docker-compose down -v
  查看后端日志：    docker logs bom_backend -f
  重启后端：       docker-compose up -d --build bom_backend

### 6.2 重新初始化数据库
  docker-compose down -v
  docker-compose up -d
  （initdb/init.sql 会自动执行）

### 6.3 备份数据库
  docker exec bom_postgres pg_dump -U bomadmin bom_system > backup.sql

## 七、开发指南

### 7.1 后端依赖
  fastapi, uvicorn, sqlalchemy, psycopg2-binary, python-jose, bcrypt, pydantic

### 7.2 修改后端代码
  后端目录通过 volume 挂载（./backend/app:/app/app），修改后执行：
  docker restart bom_backend

### 7.3 前端修改
  前端为单文件 frontend/index.html，修改后刷新浏览器即可。

## 八、架构说明

### 8.1 请求流程
  用户浏览器 -> Nginx (:80) -> FastAPI (:8000) -> PostgreSQL (:5432) / Redis (:6379)

### 8.2 端口映射
  nginx   80    ->  前端页面、API 网关
  FastAPI 8000  ->  后端 API
  postgres 5432  ->  数据库
  redis   6379   ->  缓存

## 九、常见问题

Q1: 启动后前端页面打不开？
   确认 Docker Desktop 已运行，docker ps 显示 4 个容器均为 Up。

Q2: 登录报 500 错误？
   docker logs bom_backend 查看日志，尝试 docker restart bom_backend。

Q3: 数据库连接失败？
   等待 10 秒后再试，确认 bom_postgres 容器状态为 healthy。

Q4: 修改后端代码后不生效？
   docker restart bom_backend

Q5: 忘记管理员密码？
   admin 密码默认重置为 123456。
