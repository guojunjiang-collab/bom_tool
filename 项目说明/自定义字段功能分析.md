## 一、需求目标

### 业务背景
在BOM管理系统中，零件和部件的基础字段（件号、名称、规格型号等）是固定的，无法满足不同企业/项目的特殊信息记录需求。需要提供自定义字段功能，让管理员可按需扩展零部件属性。

同时，将不常用的固定字段（材料、备注、价格、库存）移除，改由自定义字段按需添加，使系统更灵活。

### 功能范围
1. **字段定义**：管理员可定义自定义字段的名称、类型（文本/数字/单选/多选）、是否必填、选项列表
2. **数据存储**：字段定义和字段值分别存储，支持查询与筛选
3. **界面展示**：
   - 零件/部件详情页：只读展示所有自定义字段（含空值）
   - 零件/部件编辑页：可编辑的表单控件
   - 系统设置页：自定义字段定义的管理界面（增删改）
4. **子项导航**：部件详情的子项列表支持点击跳转到子项详情，可返回

## 三、详细设计方案

### 3.1 数据库设计

#### 表1：custom_field_definitions（自定义字段定义）

```sql
CREATE TABLE custom_field_definitions (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name VARCHAR(128) NOT NULL,           -- 字段显示名称，如"供应商"
    field_key VARCHAR(64) NOT NULL UNIQUE, -- 字段标识键，如"supplier"
    field_type VARCHAR(32) NOT NULL,       -- text / number / select / multiselect
    options JSONB DEFAULT '[]',            -- 单选/多选的选项列表
    is_required BOOLEAN DEFAULT FALSE,     -- 是否必填
    applies_to VARCHAR(32) NOT NULL DEFAULT 'both', -- part / component / both
    sort_order INTEGER DEFAULT 0,          -- 排序序号
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

#### 表2：custom_field_values（自定义字段值）

```sql
CREATE TABLE custom_field_values (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    field_id UUID NOT NULL REFERENCES custom_field_definitions(id) ON DELETE CASCADE,
    entity_type VARCHAR(32) NOT NULL,  -- 'part' 或 'component'
    entity_id UUID NOT NULL,           -- 零件或部件的ID
    value_text TEXT,                    -- 文本/单选值
    value_number DECIMAL(12,4),         -- 数字值
    value_json JSONB,                   -- 多选值数组
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(field_id, entity_type, entity_id)
);
```

**设计说明**：
- 字段定义与值分离，EAV模式灵活扩展
- `field_key` 作为唯一标识，避免用自增ID暴露给前端
- 值表按类型分列存储（text/number/json），便于查询和类型校验
- `applies_to` 控制字段对零件/部件/两者生效

### 3.2 后端API设计

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | /api/custom-fields/definitions/ | 获取所有字段定义（支持 ?applies_to= 筛选） | admin/engineer |
| POST | /api/custom-fields/definitions/ | 创建字段定义 | admin |
| PUT | /api/custom-fields/definitions/{id} | 更新字段定义 | admin |
| DELETE | /api/custom-fields/definitions/{id} | 删除字段定义（级联删除值） | admin |
| PUT | /api/custom-fields/definitions/reorder | 批量更新排序 | admin |
| GET | /api/custom-fields/values/{entity_type}/{entity_id} | 获取实体的所有自定义字段值 | admin/engineer |
| PUT | /api/custom-fields/values/{entity_type}/{entity_id} | 批量设置实体的自定义字段值 | admin/engineer |
| POST | /api/custom-fields/reset-data | 重置业务数据（保留用户和设置） | admin |

### 3.3 前端设计

#### 3.3.1 系统设置页

| 卡片 | 功能 |
|------|------|
| 自定义字段管理 | 表格展示字段定义，支持新增/编辑/删除 |
| 数据管理 | 导出全部数据、重置系统数据 |

**已删除：** ~~字典管理~~（材质字典已移除，改由自定义字段替代）

#### 3.3.2 零件/部件详情页

展示顺序：
1. 基础信息（件号、名称、规格型号、版本、状态）
2. 🏷️ 自定义属性（**所有适用字段全部显示，空值显示"—"**）
3. 子项列表 / 附件（部件详情）
4. 修订记录

#### 3.3.3 零件/部件编辑页

- 基础字段表单 + 自定义字段动态渲染区域
- 自定义字段控件类型：text→input / number→input / select→下拉 / multiselect→复选框组
- 保存时一并收集自定义字段值提交

#### 3.3.4 部件详情 - 子项导航

- 树形视图和表格视图的每一行可点击
- 点击零件行 → 打开零件详情
- 点击部件行 → 打开部件详情
- Footer有"← 返回部件详情"按钮，关闭子项详情后回到原部件详情

## 五、风险与应对

| 风险项 | 可能性 | 影响度 | 应对措施 |
|--------|--------|--------|----------|
| 字段定义删除后已有数据丢失 | 中 | 中 | 删除前提示确认，级联清理 |
| 大量自定义字段影响性能 | 低 | 低 | 限制字段数量上限，值表建索引 |
| 自定义字段值同步复杂度 | 中 | 中 | 与现有同步队列机制复用 |
| Docker 容器代码更新不生效 | 中 | 中 | 宿主机代码挂载后需 `docker restart` |
