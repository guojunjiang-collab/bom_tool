## 一、项目概述

### 1.1 目标

完善 BOM Tool 的前后端数据同步能力，实现所有业务字段的完整同步，并新增升版、字典管理等功能。

### 1.2 涉及文件

| 文件 | 位置 | 修改类型 |
|------|------|---------|
| `store.js` | `frontend/js/store.js` | 核心修改：字段映射、同步流程、升版逻辑 |
| `api.js` | `frontend/js/api.js` | 补充：字典 CRUD API、升版 API |
| `index.html` | `frontend/index.html` | 补充：字典管理 UI、升版 UI |
| `assemblies.py` | `backend/app/routers/assemblies.py` | 补充：升版 API |
| `parts.py` | `backend/app/routers/parts.py` | 补充：升版 API |
| `crud.py` | `backend/app/crud.py` | 补充：升版 CRUD、字典 CRUD |
| `schemas.py` | `backend/app/schemas.py` | 补充：升版 Schema、字典 Schema |
| `routers/__init__.py` | `backend/app/routers/__init__.py` | 补充：字典路由注册 |
| `main.py` | `backend/app/main.py` | 补充：字典路由挂载 |

### 1.3 实施策略

- **前端优先**：先完成前端 store.js 和 api.js 的修改，确保本地同步逻辑正确
- **前端验证**：前端修改后通过浏览器控制台日志验证同步行为
- **后端跟随**：前端验证通过后，再完善后端 API（后端修改需重启容器）
- **数据库不变**：所有修改基于现有数据库表结构，无需迁移

```python
def upgrade_part(db: Session, part_id: uuid.UUID, new_version: str) -> Part:
    """零件升版：创建新版，老版保留状态变为 superseded"""
    old_part = get_part(db, part_id)
    if not old_part:
        return None
    
    # 1. 更新老版本状态为 superseded
    old_part.status = 'superseded'
    
    # 2. 创建新版（复制所有字段，version 更新）
    new_data = PartCreate(
        code=old_part.code,

        stock=old_part.stock,
        status='draft',
```

```javascript
} else if (op === 'promote') {
  const { entity, record } = data;

  if (entity === 'parts' || entity === 'components') {
```

```python
def get_dictionaries(db: Session, dict_type: Optional[str] = None):
    query = db.query(Dictionary)
    if dict_type:
        query = query.filter(Dictionary.dict_type == dict_type)
    return query.all()

def create_dictionary(db: Session, data: DictionaryCreate):
    # 检查唯一性
    existing = db.query(Dictionary).filter(
        Dictionary.dict_type == data.dict_type,
        Dictionary.value == data.value
    ).first()
    if existing:
        raise ValueError("字典类型+值组合已存在")
    obj = Dictionary(dict_type=data.dict_type, value=data.value)
    db.add(obj)
    db.commit()
    db.refresh(obj)
    return obj

def update_dictionary(db: Session, dict_id: uuid.UUID, data: DictionaryUpdate):
    obj = db.query(Dictionary).filter(Dictionary.id == dict_id).first()
    if not obj:
        return None
    if data.value is not None:
        # 检查唯一性
        existing = db.query(Dictionary).filter(
            Dictionary.id != dict_id,
            Dictionary.dict_type == obj.dict_type,
            Dictionary.value == data.value
        ).first()
        if existing:
            raise ValueError("字典值已存在")
        obj.value = data.value
    db.commit()
    db.refresh(obj)
    return obj

def delete_dictionary(db: Session, dict_id: uuid.UUID):
    obj = db.query(Dictionary).filter(Dictionary.id == dict_id).first()
    if not obj:
        return False
    db.delete(obj)
    db.commit()
    return True
```

## 六、Phase 5：队列优化

**优先级：低** | **预估工作量：3 小时**

### 6.1 目标

### 6.2 前端修改：store.js

#### 6.2.1 操作合并

在 `_enqueue` 函数中，增加合并逻辑：

```javascript
_enqueue(op, entity, record, extra) {
  // 1. 检查队列中是否有同实体+同 ID 的待处理操作
  const existingIdx = this._syncQueue.findIndex(item => 
    item.entity === entity && item.record?.id === record?.id && item.record?.id
  );
  
  if (existingIdx >= 0) {
    const existing = this._syncQueue[existingIdx];
    if (op === 'delete') {
      // delete 优先：移除 add 或 update，保留 delete
      this._syncQueue.splice(existingIdx, 1);
      this._syncQueue.push({ op: 'delete', entity, record, extra });
      return;
    }
    if (existing.op === 'add' && op === 'update') {
      // add 后接 update：合并到 add 的 record
      Object.assign(existing.record, record);
      return;
    }
    if (existing.op === 'update' && op === 'update') {
      // 两次 update：合并字段
      Object.assign(existing.record, record);
      return;
    }
    // 否则追加新操作
  }
  
  this._syncQueue.push({ op, entity, record, extra });
}
```

#### 6.2.2 优先级队列

将 `_syncQueue` 改为带优先级的队列：

```javascript
const PRIORITY = { delete: 3, add: 2, update: 1, promote: 2, cascade: 0 };

_enqueue(op, entity, record, extra) {
  // ...合并逻辑...
  const item = { op, entity, record, extra, priority: PRIORITY[op] || 1 };
  // 按优先级插入
  const insertIdx = this._syncQueue.findIndex(q => q.priority < item.priority);
  if (insertIdx >= 0) this._syncQueue.splice(insertIdx, 0, item);
  else this._syncQueue.push(item);
}
```

#### 6.2.3 级联同步

在 `cascadeUpdateReferences` 函数中，将 cascade 操作入队：

```javascript
cascadeUpdateReferences(entity, oldId, newId) {
  // ...更新部件引用...
  if (updated) {
    this.saveAll('components', allComps);
    // 将更新后的部件入队同步
    updatedComps.forEach(c => this._enqueue('update', 'components', c));
  }
}
```

### 6.3 验证方法

1. 快速连续修改同一零件 3 次 → 队列中只有 1 个 update
2. 创建后立即删除 → 队列中只有 1 个 delete
3. 删除零件 → cascade 更新部件 → 部件 update 入队

## 九、风险与注意事项

### 9.1 风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 升版后 BOM 子项引用旧版本 | 数据不一致 | 前端升版时提示用户是否同时升版子项 |
| 字典被删除但前端缓存未更新 | 界面异常 | 字典缓存增加版本号，变更时刷新 |
| 字段映射错误导致数据覆盖 | 数据损坏 | 上线前完整测试所有字段 |
| 队列合并导致某些操作丢失 | 功能异常 | 严格测试合并边界条件 |

### 9.2 注意事项

1. **后端修改需重启容器**：`docker restart bom_backend`
2. **数据库无变更**：所有修改基于现有表结构，无需迁移
3. **前端热更新**：修改 `index.html` 后刷新浏览器即可
4. **store.js 热更新**：需刷新浏览器（Docker volume 已挂载，需重启：`docker restart bom_backend`）
5. **测试时清空队列**：浏览器控制台执行 `Store._syncQueue = []; Store.saveAll('parts', Store.getAll('parts'))` 可清空待同步队列
