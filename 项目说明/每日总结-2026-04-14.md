## 🚀 主要功能实现

### 1. BOM反查界面全面重构
**目标**：优化BOM反向追溯（引用树）界面的用户体验

#### 1.1 树形节点显示优化
- **移除图标**：删除原有的 📦（部件）/🔩（零件）emoji图标，界面更简洁
- **层级前置**：层级标识（`L0`/`L1`/`L2`...）从行末尾移至最前面，采用蓝色圆角徽章样式
- **层级计数修正**：根节点（被查零件本身）从 L0 开始，直接父项 L1，逐级递增
- **内容精简**：每条节点仅显示件号、名称、版本、状态，移除用量（×qty）显示
- **版本显示**：去V前缀，直接显示版本字母如 `A`、`B`、`C`

**树形节点字段对照**：
| 字段 | 说明 |
|------|------|
| 件号 | `node.code`，无则显示 `-` |
| 名称 | `node.name`，无则显示 `?` |
| 版本 | `node.version`，无则不显示 |
| 状态 | `node.status`，无则不显示（使用 `UI.statusTag` 标签） |
| 层级 | `L0` 起始，蓝色圆角徽章，位于最前面 |

#### 1.2 搜索交互改造
- **下拉框 → 搜索框**：将原来的 `<select>` 下拉选择改为文本输入框，placeholder 为"输入件号或名称搜索零件..."
- **实时筛选**：`input` 事件驱动，实时在零件库中过滤匹配结果
- **筛选逻辑**：支持件号（`code`）和名称（`name`）的中英文模糊匹配，大小写不敏感
- **搜索列表样式**：
  - 表格形式，表头：件号、名称、版本、状态
  - 鼠标悬停高亮（`#f0f7ff`）
  - 滚动容器，最大高度 320px
- **点击选中**：点击行直接触发 `doReverse(pid)` 查询，结果树替换搜索区域
- **容器显示控制**：搜索列表容器初始 `display:none`，仅在有搜索词且有结果时显示，无匹配时显示提示文本

#### 1.3 父项查询逻辑修复
**问题**：原查询逻辑中 `parent_type === 'component'` 条件永远为 false（后端实际存储值为 `'assembly'`），导致所有父项查找均失败

**修复**：
```javascript
// 修改前（错误）
var parentEntity = pt === 'component'
  ? comps.find(function(c){return c.id === puid;})
  : partsAll.find(function(p){return p.id === puid;});

// 修改后（正确）
var parentEntity = (pt === 'assembly' || pt === 'component')
  ? comps.find(function(c){return c.id === puid;})
  : partsAll.find(function(p){return p.id === puid;});
```

**同时修复递归条件**：原 `if (pt === 'assembly')` 改为 `if (pt === 'assembly' || pt === 'component')`，确保 assembly 类型继续向上追溯

### 3. 前端BOM items同步机制
**目标**：将 `bom_items` 数据纳入前端 Store 的同步体系，实现登录后自动拉取完整BOM关系

**新增同步项**：在 `Sync._run()` 的 entities 列表中新增 `bom_items`，使登录后同步阶段自动拉取全部BOM items数据到本地 Store

**字段映射修复**：`Store._fieldMap['bom_items']` 补全关键字段：
```javascript
bom_items: {
  childId: 'child_id',    // 修复：原错误映射为 'id'（BOMItem自身UUID）
  parentId: 'parent_id',
  parentType: 'parent_type',
  partId: 'part_id',
  quantity: 'quantity'
}
```

**API接口**：前端 `API.getAllBomItems()` 方法，对应 `GET /api/bom/items/all`

## 📊 技术架构更新

### BOM反查数据流（修复后）
```
用户输入搜索词
    ↓
_filterPartList(keyword)
    ↓ 从 Store.getAll('parts') 过滤匹配
    ↓
点击零件行 → doReverse(pid)
    ↓
从 Store.getAll('bom_items') 构建 childToParents 索引
    ↓ child_id → [bomItem, ...]
从 Store.getAll('components') / Store.getAll('parts') 查找父项
    ↓ parent_type='assembly' → components表
    ↓
构建树形结构 { id, code, name, version, status, level, parents }
    ↓
_renderTreeNode() 渲染
    ↓
树形视图：L0(根) → L1(父) → L2(祖父) → ...
```
